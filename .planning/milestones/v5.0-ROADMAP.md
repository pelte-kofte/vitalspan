# Milestone v5.0: Personalization & Production

**Status:** ✅ SHIPPED 2026-06-19
**Phases:** 20-23
**Total Plans:** 15

## Overview

v5.0 turns users from passive data viewers into active longevity managers. The milestone delivers a personal exercise routine with drag-to-reorder and progressive overload tracking, a fully overhauled editable protocol with adherence streaks and AM/PM/Evening/Night push notification reminders, biomarker trend sparklines with range-band overlays, free-tier 30-day data limits, and a production EAS build shipped to TestFlight. The AI Advisor context is enriched with personal dose bucketing (high/standard/low) for pharmacist-safe recommendations.

## Milestone Summary

**Key Decisions:**

- ProtocolItem unifies addedSupplements + customSupplements with source discriminant ('db' | 'manual') — CustomSupplement retained migration-detection-only
- hiddenMeds: string[] on ProtocolState for soft-hide of medications from protocol view
- react-native-draggable-flatlist used for routine drag-to-reorder
- react-native-chart-kit used for sparklines (victory-native excluded — Skia peer conflict with Expo SDK 54)
- Personal dose stored AsyncStorage-only for v5.0 — Supabase sync deferred
- Raw personalDose string omitted from AI context (bucketed high/standard/low only, pharmacist liability)
- expo-notifications chosen for local push (remote push deferred to v5.1+)
- Module-scope Notifications.setNotificationHandler (SDK 54 shouldShowBanner/shouldShowList API) — iOS foreground banner pattern

**Issues Resolved:**

- ProtocolState "Custom" category confusion — replaced with type-correct supplements[] and medications[] sections via migrateProtocol()
- ExerciseLogEntry lacked weightKg/repsPerSet — added SetRecord interface and setsData[] field
- REQUIREMENTS.md NTFY/PROD checkboxes left unchecked after Phase 23 execution — corrected at milestone close

**Issues Deferred:**

- Remote push notifications via Supabase push tokens — v5.1+
- Multiple named routines (Upper Body / Leg Day) — v6.0+
- AI-generated routine recommendations — v6.0+
- Articles redesign (card/grid layout, search, AI personalization) — v5.1
- Personal dose Supabase sync — future milestone

**Technical Debt Incurred:**

- Phase 21 row in ROADMAP.md progress table showed 0/5 (not updated after execution) — corrected at milestone close
- STATE.md progress metrics partially stale (velocities not tracked for Phases 22-23)
