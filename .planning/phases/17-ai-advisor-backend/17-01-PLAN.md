Table name: ai_usage
Columns:
  - user_id   uuid      NOT NULL  REFERENCES auth.users(id) ON DELETE CASCADE
  - date       date      NOT NULL
  - report_count  int   NOT NULL  DEFAULT 0
  - chat_count    int   NOT NULL  DEFAULT 0
  - PRIMARY KEY (user_id, date)

Limits (D-07):
  - report_count max: 5 per (user_id, date)
  - chat_count max: 20 per (user_id, date)

RLS note (D-16):
  - Edge Function runs as service_role — bypasses RLS; no RLS SELECT needed for Edge Function.
  - Client-side: authenticated users may SELECT their own row (for future UI display of remaining quota).
  - Client-side: NO INSERT/UPDATE from client — only Edge Function modifies counts.
