<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Expo app → Edge Function | JWT in Authorization header; function must verify before any processing |
| Edge Function → Anthropic API | API key secret; must never appear in client bundle |
| Edge Function → ai_usage table | Service role write; function is sole authority on rate counts |
| Context payload → Claude | Anonymized context; no PII should reach Anthropic (enforced by advisorContext.ts) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-17-08 | Spoofing | JWT verification | mitigate | `serviceClient.auth.getUser(token)` validates JWT signature via Supabase; invalid tokens → immediate 401 |
| T-17-09 | Tampering | Request body | mitigate | action field validated to 'report'|'chat' before processing; context is anonymized by client |
| T-17-10 | Repudiation | Rate limit bypass | mitigate | Rate check happens before Anthropic call; counter incremented after success; race window acceptable for this use case (see note in Task 1) |
| T-17-11 | Information Disclosure | Anthropic API key | mitigate | Key read-only via `Deno.env.get('ANTHROPIC_API_KEY')` — never in source code; set via `supabase secrets set`; never logged |
| T-17-12 | Denial of Service | Anthropic API spend | mitigate | Rate limits (5/20 per day per user) are primary spend guard; `max_tokens` cap (2000 report, 500 chat) bounds per-call cost |
| T-17-13 | Elevation of Privilege | Anonymous user access | mitigate | JWT verification blocks anonymous session IDs — per D-16, anonymous users never reach the Edge Function (blocked by premium gate in Phase 16); if they did, getUser() would return an anon user which is still a valid user — accept this risk as premium gate handles it |
| T-17-14 | Information Disclosure | Error messages | mitigate | Error responses return generic strings only — no stack traces, no Supabase internals exposed |
| T-17-SC | Tampering | npm/pip/cargo installs | accept | Deno uses URL imports, no package manager; no install step required |
</threat_model>
