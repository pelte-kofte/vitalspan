## Asset Specifications

### ASSET-01: App Icon

| Property | Value |
|----------|-------|
| File | `assets/icon.png` |
| Required dimensions | 1024 × 1024px PNG |
| Current state | 1 × 1px placeholder (confirmed via `file` command) |
| Background fill | `#2D6A4F` (solid — `Colors.primary`) |
| Foreground mark | White lettermark — single capital "V" in DM Serif Display, weight 400, centered |
| Mark size | ~480px cap height (fills ~47% of canvas vertically — standard icon safe zone) |
| Mark color | `#FFFFFF` |
| Corner rounding | None — iOS applies its own superellipse mask; do not pre-round the PNG |
| Generation method | Node.js script at project root: `scripts/generate-icon.js` using `canvas` npm package |

**Why lettermark, not wordmark:** "Vitalspan" is too long to legibly render at 1024px when scaled to 60px on a home screen grid. A single "V" on `#2D6A4F` is immediately recognizable and consistent with the green palette.

**Generation script contract (`scripts/generate-icon.js`):**
```
canvas 1024×1024
fill rect: #2D6A4F
font: DM Serif Display (or fallback serif) at 480px
fillStyle: #FFFFFF
textAlign: center / textBaseline: middle
fillText('V', 512, 512)
output: assets/icon.png
```

The script must be runnable with `node scripts/generate-icon.js`. After running, `app.json` `icon` field already points to `./assets/icon.png` — no config change needed.

#### Change 5 — STORE-03: Legal section with disclaimer acceptance date

**What changes:** A new "Legal" section card is added immediately after the existing medical disclaimer card (`s.disclaimer`) and before the credits footer. This is separate from the existing `disclaimerTitle` card (which stays unchanged).

**New "Legal" section card layout:**

```
┌─────────────────────────────────────────────────────┐
│  Legal                                               │
│                                                      │
│  Medical disclaimer v1.0                             │
│  Accepted: [date or "Not yet accepted"]              │
│                                                      │
│  App version: {VERSION}                              │
└─────────────────────────────────────────────────────┘
```

**Data source:**
- AsyncStorage key: `@vitalspan_disclaimer_accepted` (confirmed from MedicalDisclaimer.tsx line 8)
- Stored format: `JSON.stringify({ version: string, acceptedAt: string })` — where `acceptedAt` is an ISO 8601 string (confirmed from MedicalDisclaimer.tsx line 55)
- Read with `AsyncStorage.getItem('@vitalspan_disclaimer_accepted')` — parse with `JSON.parse`
- Display format for `acceptedAt`: use `new Date(acceptedAt).toLocaleDateString('en-US', { year: 'numeric', month: 'long', day: 'numeric' })` — e.g. "May 25, 2026"

**State needed in AboutScreen:**
```typescript
const [disclaimerInfo, setDisclaimerInfo] = useState<{
  version: string;
  acceptedAt: string;
} | null>(null);

useEffect(() => {
  AsyncStorage.getItem('@vitalspan_disclaimer_accepted')
    .then(raw => {
      if (raw) setDisclaimerInfo(JSON.parse(raw) as { version: string; acceptedAt: string });
    })
    .catch(() => {});
}, []);
```

**Exact copy contract — "Legal" section:**

| Disclaimer accepted | Accepted date line | "Accepted: {formattedDate}" |
| Disclaimer NOT accepted | Disclaimer line | "Medical disclaimer v1.0" |
| Disclaimer NOT accepted | Accepted date line | "Not yet accepted" |
| App version line | Always | "App version: {VERSION}" |
