## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | biomarker_entries RLS | mitigate | INSERT policy: `WITH CHECK (user_id = auth.uid())` — anonymous session provides auth.uid(); client cannot insert rows for other users |
| T-08-02 | Information Disclosure | biomarker_entries SELECT | mitigate | SELECT policy: `USING (user_id = auth.uid())` — users can only read their own rows; cross-session data leakage is blocked at the database level |
| T-08-03 | Tampering | No DELETE/UPDATE RLS | accept | Table is append-only by design (D-08); no DELETE or UPDATE policy means the client cannot modify or remove rows even if a malicious actor attempts it |
| T-08-SC | Tampering | npm/pip installs | accept | No new packages installed in this plan; schema-only SQL file |
</threat_model>
