    Internal `searchPubMed(query: string, retmax = 5): Promise<string[]>` — calls eSearch with `db=pubmed&term=encodeURIComponent(query)&retmax=${retmax}&retmode=json`, parses `json.esearchresult.idlist`. Type the response as `{ esearchresult: { idlist: string[] } }`.

    Internal `fetchAbstract(pmid: string): Promise<string | null>` — calls eFetch with `db=pubmed&id=${pmid}&rettype=abstract&retmode=xml`, text response, regex `/<AbstractText[^>]*>([\s\S]*?)<\/AbstractText>/` to extract content, strips any remaining XML tags with `replace(/<[^>]+>/g, '')`. Returns null if no match.

    Internal `fetchAllBiomarkerArticles(): Promise<Article[]>` — iterates over all entries in BIOMARKER_QUERIES sequentially (not parallel — rate limit protection per Pitfall 5). For each query: call `searchPubMed(query, 5)` → collect up to 5 PMIDs. After all eSearch calls complete, batch all unique PMIDs into one eSummary call: `${ESUMMARY_BASE}?db=pubmed&id=${pmids.join(',')}&retmode=json`. Parse eSummary response typed as `{ result: Record<string, { uid: string; title: string; fulljournalname: string; pubdate: string }> }`. Then for each PMID (sequentially), call `fetchAbstract(pmid)` with a 350ms delay between calls (use `await new Promise(r => setTimeout(r, 350))`). Assemble Article objects from eSummary metadata + abstract. Set `biomarker_tags` to the biomarker ID key(s) that returned this PMID from eSearch — track the pmid→biomarkerKey mapping during eSearch phase. Set `fetched_at: new Date().toISOString()`. Return the assembled array.

    Export `loadCachedArticles(entries: StoredEntry[]): Promise<Article[]>` — selects all from Supabase articles table (`select('*')`), returns `rankByOutOfRange(data, entries)` or `[]` on error. Wrap in try/catch with `console.error`.

    Export `refreshArticlesIfStale(entries: StoredEntry[]): Promise<Article[] | null>` — checks `@vitalspan_articles_last_fetched` from AsyncStorage. If age < CACHE_TTL_MS, return null immediately. Never throws — all errors caught with `console.error`, return null on failure.

    Export `forceRefreshArticles(entries: StoredEntry[]): Promise<Article[] | null>` — same as refreshArticlesIfStale but skips the staleness check (used by ArticlesScreen pull-to-refresh).

Build `outOfRange` Set of biomarker IDs where `entry.value < bm.optMin || entry.value > bm.optMax` using `BIOMARKERS[].optMin` and `BIOMARKERS[].optMax`. Sort articles: articles with any `biomarker_tags` item in `outOfRange` sort first. Secondary sort: articles with more matching out-of-range tags sort higher. Stable sort (does not reorder equal items).

    TypeScript strict: no `any`. All NCBI JSON response shapes explicitly typed with local interfaces (NCBIESearchResponse, NCBISummaryResponse, NCBISummaryArticle). File max 200 lines — keep function bodies concise.
