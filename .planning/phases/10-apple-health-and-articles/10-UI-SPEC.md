# Phase 10 — UI Design Contract: Apple Health + Articles

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

## Spacing Scale

All spacing uses `Spacing.*` tokens from `src/theme/index.ts`. No hardcoded margin/padding numbers in new files.

| Token | Value | Usage in This Phase |
|-------|-------|---------------------|
| `Spacing.xs` | 4 | Icon gaps, badge inner padding |
| `Spacing.sm` | 8 | Row gap between label + value, compact vertical rhythm |
| `Spacing.md` | 12 | Card inner padding (compact), chip padding |
| `Spacing.base` | 16 | Card horizontal padding, screen horizontal margins |
| `Spacing.lg` | 20 | Section vertical gap |
| `Spacing.xl` | 24 | Card stack gap, list item separator breathing room |
| `Spacing.xxl` | 32 | Screen bottom padding |

Exceptions:
- Minimum touch target: 44px height on all tappable rows (ArticleCard, permission prompt CTAs). Use explicit `height: 44` or `minHeight: 44` on the touchable — do not use a Spacing token for touch target minimums since none map to 44.
- Orbital orb size: `width: 60, height: 44` (inherited from LongevityScoreScreen existing `dataOrb` style — do not change).
- Permission prompt card: `paddingVertical: 20` (= Spacing.lg) to give it visual weight distinct from info cards.

> Source: `src/theme/index.ts` Spacing export; LongevityScoreScreen `s.dataOrb` style; CLAUDE.md spacing rule.

## Component Inventory

### New Components (must be created)

#### 1. `ArticleCard` — `src/components/ArticleCard.tsx`

A touchable card displaying one PubMed article. Max 200 lines — extract from ArticlesScreen if needed.

**Visual spec:**
- Container: `backgroundColor: Colors.bgCard`, `borderRadius: Radius.xl` (20), `marginHorizontal: Spacing.base`, `marginBottom: Spacing.sm`, `padding: Spacing.base`
- Shadow: `Elevation.sm` from theme (`shadowOpacity: 0.06, shadowRadius: 8, elevation: 2`)
- Layout: vertical stack — [eyebrow row] → [title] → [abstract snippet] → [meta row]
- **Eyebrow row**: `Colors.Beige.textMuted` (`#6B6B64`) journal name at `Typography.sizes.sm` (12px) weight 400, left-aligned. Publication date at same size, right-aligned. Row uses `flexDirection: 'row', justifyContent: 'space-between'`.
- **Title**: `Typography.sizes.base` (14px) weight 600, `Colors.textPrimary` (`#1A1A18`), `lineHeight: 20`, `marginTop: Spacing.xs` (4), max 2 lines (`numberOfLines: 2, ellipsizeMode: 'tail'`).
- **Abstract snippet**: `Typography.sizes.body` (15px) weight 400, `Colors.textSecondary` (`#4A4A45`), `lineHeight: 22`, `marginTop: Spacing.sm` (8), max 3 lines (`numberOfLines: 3, ellipsizeMode: 'tail'`).
- **Meta row**: `marginTop: Spacing.sm` (8), `flexDirection: 'row', alignItems: 'center', justifyContent: 'space-between'`. Left side: relevance tag (if article is ranked for an out-of-range biomarker). Right side: "Read →" in `Colors.primary` (`#2D6A4F`), `Typography.sizes.sm` (12px) weight 600.
- **Relevance tag** (shown only when article surfaces for an out-of-range biomarker): pill shape, `backgroundColor: Colors.status.reviewBg` (`#FEF9E7`), `borderColor: Colors.status.reviewBorder`, `borderWidth: 0.5`, `borderRadius: Radius.full`, `paddingHorizontal: 8, paddingVertical: 2`. Text: biomarker name in `Typography.sizes.xs` (11px) weight 600, `Colors.status.reviewText` (`#7A5B10`).
- Touch: `TouchableOpacity`, `activeOpacity: 0.82`, haptic: `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Light)` on press.

#### 2. Permission prompt card — inline in `LongevityScoreScreen.tsx` (not a separate file)

Three states replace the current `connectAndSync` CTA area at the bottom of LongevityScoreScreen. States are driven by `permissionState: 'pre-request' | 'granted' | 'denied'` derived from AsyncStorage `@vitalspan_health_permissions`.

**State A — Pre-request (first visit, not yet asked)**

Shown when `hasRequestedHealthKit` is not set in AsyncStorage.

- Container: `backgroundColor: 'rgba(20,25,22,0.85)'`, `borderWidth: 0.5`, `borderColor: Colors.dark.border`, `borderRadius: Radius.xl`, `marginHorizontal: Spacing.base`, `padding: Spacing.lg` (20), `marginBottom: Spacing.base`.
- Icon: `'⌚'` glyph, `fontSize: 28`, `marginBottom: Spacing.sm`.
- Headline: "Connect Apple Health" — `Typography.sizes.base` (14px), weight 600, `Colors.dark.text`.
- Body: "Sync HRV, sleep, VO₂ max and glucose to power your longevity orbitals." — `Typography.sizes.body` (15px), weight 400, `Colors.dark.textMuted`, `lineHeight: 22`, `marginTop: Spacing.xs`.
- CTA button: full-width, `backgroundColor: Colors.primaryLight` (`#52B788`), `borderRadius: Radius.xl`, `height: 44`, `marginTop: Spacing.md` (12). Label: "Connect Apple Health" — `Typography.sizes.base` (14px), weight 600, `Colors.dark.bg` (`#0C0F0D`).
- Haptic on CTA tap: `Haptics.impactAsync(Haptics.ImpactFeedbackStyle.Medium)`.

**State B — Granted**

Existing `s.healthKitCard` / `s.healthKitCardConnected` pattern is preserved. The `isDemoMode` badge disappears automatically when real data has no `isDemoMode` field. No new UI elements required — existing component remains.

**State C — Denied (permissions declined on iOS system prompt)**

Shown when empty reads are detected after `initHealthKit` resolves (iOS cannot report denial directly — detect via empty HRV + sleep result after init).

- Container: same dimensions as State A.
- Icon: `'⚠'` glyph (not emoji), `fontSize: 20`, `color: Colors.warning` (`#D4860B`), `marginBottom: Spacing.sm`.
- Headline: "Apple Health access needed" — same as State A headline style.
- Body: "Enable Health access in Settings to see live HRV, sleep and recovery data." — same style.
- CTA button: same dimensions as State A. Label: "Open Settings" — same label style.
- CTA action: `Linking.openURL('app-settings:')` — no navigation.
- Secondary link below CTA: "Continue without Health data" in `Colors.dark.textMuted`, `Typography.sizes.sm` (12px), centered, `marginTop: Spacing.sm`. Tap dismisses the card (user proceeds with manual biomarker data only).

#### 3. Research CTA card — inline in `DashboardScreen.tsx`

### New Screens

#### `ArticlesScreen` — `src/screens/ArticlesScreen.tsx`

**Layout:**
- `SafeAreaView` with `backgroundColor: Colors.bg` (`#EDE8DC`) — matches warm screens.
- `StatusBar` style: `dark-content` (warm background, same as Biomarkers/Protocol screens).
- Top bar: back chevron `←` at left (`TouchableOpacity`, `onPress: nav.goBack()`), screen title "Research" centred at `Typography.sizes.xs` (11px) weight 600, `Colors.textMuted`, `letterSpacing: Typography.letterSpacing.widest` (3). No right action.
- `FlatList` below top bar, full height. `contentContainerStyle: { paddingTop: Spacing.base, paddingBottom: Spacing.xxl }`.
- `ItemSeparatorComponent`: none — `ArticleCard` uses `marginBottom: Spacing.sm` to create gap.
- `keyExtractor`: `(item) => item.pmid`.

**Loading state:**
- When Supabase returns no cached data yet: show a centred `ActivityIndicator` in `Colors.primary`. No skeleton — app is offline-first so empty cache is edge case (first-ever open before background fetch).
- When cached data present but background refresh in progress: show cached list immediately. No spinner overlay — do not block the list.

**Empty state** (cache is empty and fetch failed):
- `View` centred vertically in the list area, `paddingHorizontal: Spacing.xxl`.
- Body text: "No articles available. Check your connection and pull to refresh." — `Typography.sizes.body` (15px), `Colors.textMuted`, `textAlign: 'center'`, `lineHeight: 22`.
- No icon, no CTA button.

**Pull-to-refresh:**
- `RefreshControl` on FlatList, `tintColor: Colors.primary`, manually triggers `articleService.refreshArticlesIfStale` ignoring the 24h gate (force refresh on pull).

## Interaction Contracts

### LongevityScoreScreen — Permission State Transitions

```
Mount → AsyncStorage.getItem('@vitalspan_health_permissions')
  ├── key absent or { hasRequestedHealthKit: false }
  │     → render State A (pre-request prompt card)
  │     → [user taps "Connect Apple Health"]
  │           → haptic Medium
  │           → setConnecting(true)
  │           → call initHealthKit() [react-native-health]
  │           → on callback: attempt HRV read
  │                 ├── HRV result non-empty → setPermissionState('granted')
  │                 │     → connectAndSync() → populate orbitals
  │                 │     → save { hasRequestedHealthKit: true, granted: true }
  │                 └── HRV result empty → setPermissionState('denied')
  │                       → save { hasRequestedHealthKit: true, granted: false }
  │                       → render State C (denied prompt)
  │
  ├── { hasRequestedHealthKit: true, granted: true }
  │     → loadHealthData() → render live orbitals (State B — existing)
  │
  └── { hasRequestedHealthKit: true, granted: false }
        → render State C (denied prompt)
        → [user taps "Open Settings"] → Linking.openURL('app-settings:')
        → [user taps "Continue without Health data"] → dismiss card, show empty orbitals
```

### ArticlesScreen — Load Sequence

```
Mount
  → loadCachedArticles(userEntries) from Supabase
        ├── data present → render ranked ArticleCard list
        └── data empty → show ActivityIndicator
  → refreshArticlesIfStale(userEntries) [background, non-blocking]
        ├── cache fresh (< 24h) → no-op
        └── cache stale → fetch NCBI → upsert Supabase → re-render list

Pull-to-refresh
  → force-refresh (ignore 24h gate)
  → show RefreshControl spinner
  → re-render list when done

ArticleCard tap
  → haptic Light
  → WebBrowser.openBrowserAsync(`https://pubmed.ncbi.nlm.nih.gov/${pmid}/`)
```

### Profile/Settings — Disconnect Health (HK-04)

No new UI component required beyond a destructive row in existing ProfileScreen. The visual contract:
- Row label: "Disconnect Apple Health" in `Colors.danger` (`#B13126`), `Typography.sizes.base` (14px) weight 400. Shown only when `granted: true` in `@vitalspan_health_permissions`.
- Tap: `Alert.alert('Disconnect Apple Health', 'Orbitals will revert to manual-entry data. You can reconnect anytime.', [{ text: 'Cancel', style: 'cancel' }, { text: 'Disconnect', style: 'destructive', onPress: handleDisconnect }])`.
- `handleDisconnect`: clears `@vitalspan_health_permissions` and `@vitalspan_health_data`, sets `permissionState` to `'pre-request'` in LongevityScoreScreen on next focus.

## Animation Contracts

### LongevityScoreScreen — Permission Prompt Card

- The prompt card fades in using `withTiming(1.0, { duration: 400 })` on `opacity` — matches `dpOpacity` entrance animation duration.
- No scale entrance on the prompt card (keep the animation system simple; the sphere already animates).
- The `connectAndSync` loading state on State A CTA button: replace button label with "Connecting…" and disable the button (`disabled: true`) while `connecting === true`. No spinner inside the button — text change is sufficient.

### ArticlesScreen — No Custom Animation

- Use standard `FlatList` without animated entrance. This screen is informational — avoid adding animation complexity.
- `RefreshControl` uses platform default pull-to-refresh spinner.

## Accessibility

- All tappable elements: minimum `44 × 44` touch target. `ArticleCard` achieves this via `minHeight: 44` on the touchable wrapper.
- `accessibilityRole="button"` on all `TouchableOpacity` elements in new UI.
- `accessibilityLabel` on icon-only back chevron: `accessibilityLabel="Go back"`.
- `accessibilityLabel` on ArticleCard: `accessibilityLabel={`${title}, ${journal}, published ${pubDate}`}`.
- Permission prompt CTAs: `accessibilityLabel="Connect Apple Health"` and `accessibilityLabel="Open Settings"`.
- `numberOfLines` + `ellipsizeMode` on title (2 lines) and abstract (3 lines) — required to prevent unbounded card height on long academic titles.
