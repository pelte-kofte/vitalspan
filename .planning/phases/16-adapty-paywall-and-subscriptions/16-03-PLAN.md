<task type="checkpoint:human-action">
  <name>Task 1: Adapty Dashboard + App Store Connect Setup</name>
  <what-to-do>
    Complete all required external configuration. No code changes are made in this task.

    PART A — App Store Connect Products (apps.apple.com/account/developer):
    1. Log in to App Store Connect → Apps → Vitalspan → Monetization → Subscriptions
    2. Create a Subscription Group (e.g., "Vitalspan Premium") if one does not exist
    3. Create the annual subscription product:
       - Product ID: com.vitalspan.app.premium.annual
       - Reference Name: Vitalspan Premium Annual
       - Subscription Duration: 1 Year
       - Price: set a price tier (e.g., $79.99/yr — you can adjust later)
       - Add an Introductory Offer: type "Free Trial", duration "7 Days"
    4. Create the monthly subscription product (same subscription group):
       - Product ID: com.vitalspan.app.premium.monthly
       - Reference Name: Vitalspan Premium Monthly
       - Subscription Duration: 1 Month
       - Price: set a price tier (e.g., $9.99/mo)
       - No introductory offer on monthly (per D-07)
    5. Save both products (they do not need to be in "Ready to Submit" state for Adapty sandbox testing)

    PART B — Adapty Dashboard (app.adapty.io):
    1. Create an Adapty account if you don't have one (free tier is sufficient for development)
    2. Create a new app: App Name "Vitalspan", Bundle ID "com.vitalspan.app", Platform iOS
    3. Go to App Settings → iOS SDK → Add App Store Connect API key (generate in ASC under Users and Access → Integrations → App Store Connect API)
    4. Go to Products → Import products from App Store Connect (or create manually matching the two product IDs above)
    5. Go to Access Levels — verify "premium" access level exists (auto-created). If it does not exist, create one named exactly "premium" (lowercase)
    6. Assign both products to the "premium" access level
    7. Go to Paywalls → Create Paywall → Add both products to the paywall
    8. Go to Placements → Create Placement:
       - Placement ID: vitalspan_premium_paywall
       - Link the paywall you just created to this placement
    9. Save and Publish the placement (CRITICAL: adapty.getPaywall() returns nothing until published)
    10. Copy the Public SDK Key from App Settings → API Keys → Public SDK key

    PART C — Environment Setup:
    1. Create a .env file in the project root (if it does not exist — it is gitignored)
    2. Add: EXPO_PUBLIC_ADAPTY_API_KEY=<paste the public SDK key from step B-10>
    3. Verify .env is in .gitignore (should already be)

    PART D — Sandbox Apple ID:
    1. In App Store Connect → Users and Access → Sandbox → Sandbox Testers
    2. Create a sandbox tester Apple ID (use a different email than your real Apple ID)
    3. Note the sandbox email and password — needed for testing purchases on device

    PART E — Prebuild (native module installation):
    1. In your terminal at the project root: npx expo prebuild --clean
    2. Then: cd ios && pod install && cd ..
    3. Then rebuild the dev client: npx expo run:ios
    Note: --clean regenerates the ios/ folder. All config plugins (HealthKit, Adapty) are re-applied automatically from app.json. Do not manually edit files in ios/ after this.
  </what-to-do>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App Store Connect → Adapty | API key authorizes Adapty to read product info from ASC; this key must remain in Adapty dashboard only — never in source code |
| Adapty → StoreKit | Adapty calls Apple's receipt validation servers; purchase confirmation is server-side |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-16-04 | Information Disclosure | App Store Connect API key | mitigate | ASC API key is stored in Adapty dashboard settings only — never added to source code or .env. Only the public Adapty SDK key goes in .env. |
| T-16-05 | Tampering | Fake restore (receipt spoofing) | mitigate | adapty.restorePurchases() calls StoreKit → Apple validates receipt server-side → Adapty only grants access if Apple confirms. No client-side bypass possible. |
