<objective>
Fix concrete layout issues found during form-factor audit (POLISH-01). Three targeted fixes:

1. **AppNavigator tab bar** — The `paddingBottom: 8` in tabBarStyle is a hardcoded value that ignores the device's safe area inset. On iPhone 16 Plus (bottom inset 34px), React Navigation adds its own safe area handling but the fixed paddingBottom can cause overlap. Replace with `useSafeAreaInsets` from `@react-navigation/native` to set `paddingBottom` dynamically: `Math.max(insets.bottom, 8)`.

2. **LandingScreen** — The `inner` View uses `justifyContent: 'center'` with no ScrollView. On iPhone SE or phones with large dynamic type, the 56px serif title + feature list + CTA stack can overflow SafeAreaView. Wrap the `inner` View content in a ScrollView with `contentContainerStyle={{ flexGrow: 1, justifyContent: 'center' }}` so content centers when it fits and scrolls when it doesn't.

3. **OnboardingScreen** — Steps 0 (name TextInput, autoFocus) and 4 (medication step with MedicationSearch) have a `ScrollView` + outer `View` CTA pattern where the keyboard obscures the CTA button. Wrap the per-step SafeAreaView children in a `KeyboardAvoidingView` with `behavior={Platform.OS === 'ios' ? 'padding' : 'height'}` and `style={{ flex: 1 }}`. This applies to all 5 steps since they share the same pattern structure (step 0 has autoFocus, step 4 has MedicationSearch).

Purpose: All screens render correctly on iPhone 15 Pro and iPhone 16 Plus — no layout breaks block TestFlight submission.
Output: Three files with targeted, non-breaking layout fixes. No visual design changes. No logic changes.
</objective>
