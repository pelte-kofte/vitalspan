## Spacing Scale

Source: `src/theme/index.ts` `Spacing` export — pre-populated, no changes.

| Token | Value | Usage |
|-------|-------|-------|
| `Spacing.xs` | 4px | Icon gaps, chip padding |
| `Spacing.sm` | 8px | Inline gaps, section header bottom margin |
| `Spacing.md` | 12px | Card internal row padding (vertical) |
| `Spacing.base` | 16px | Screen horizontal padding, card horizontal margin |
| `Spacing.lg` | 20px | Hero section vertical padding |
| `Spacing.xl` | 24px | Empty state internal padding |
| `Spacing.xxl` | 32px | Bottom scroll buffer |

Exceptions:
- List separator: `borderBottomWidth: 0.5` — not a spacing value, it is a visual weight
- Touch targets (CTA buttons, empty state CTA): minimum height 44px — iOS HIG minimum
- Tab bar height: `Math.max(insets.bottom, 0) + 56` — already set in AppNavigator, do not change

## Typography

Source: `src/theme/index.ts` `Typography.sizes` — pre-populated.

| Role | Token | Size | Weight | Line Height | Usage |
|------|-------|------|--------|-------------|-------|
| Screen title / Display | `Typography.sizes.xxl` | 28px | 700 | 1.2 | Screen-level headings (Exercise, Biomarkers, Profile) |
| Section header / Label | 11px (hardcoded per existing convention) | 11px | 600 | 1.0 | `sectionLabel` — UPPERCASE, letterSpacing 1.5 |
| Body / Row primary | `Typography.sizes.base` | 14px | 400–500 | 1.5 | Card row text, list item primary |
| Caption / Meta | `Typography.sizes.xs` | 11px | 400 | 1.4 | Subtitles, timestamps, units |

Rules:
- Only four size steps above are used. No ad hoc `fontSize` values.
- Section labels must always be `textTransform: 'uppercase'` + `letterSpacing: 1.5` + weight 600.
- Body line height must be set explicitly: `lineHeight: 21` at 14px (1.5 ratio).
- Caption line height: `lineHeight: 16` at 11px.
- No new font sizes may be introduced in this phase.

## Status Bar Contract (THEME-04)

Implementation pattern: `useFocusEffect` + `expo-status-bar` `setStatusBarStyle` call.

```tsx
// Pattern for ALL warm screens
import { useFocusEffect } from '@react-navigation/native';
import { setStatusBarStyle } from 'expo-status-bar';
import React, { useCallback } from 'react';

useFocusEffect(
  useCallback(() => {
    setStatusBarStyle('dark');
    return () => {
      // No cleanup needed — next screen sets its own style
    };
  }, [])
);
```

| Screen group | StatusBar style | Background context |
|-------------|-----------------|-------------------|
| Warm screens (Biomarkers, Protocol, Exercise, Profile, Settings, About, BiomarkerEntry, BiomarkerDetail) | `'dark'` | Light cream — dark text needed |
| Dark screens (Dashboard, LongevityScore, Landing) | `'light'` | Dark neural — light icons needed |

Notes:
- Each screen sets its own style on focus — no centralized controller needed.
- Dashboard, Landing, and LongevityScore already manage dark status bar. Do not touch them.
- BiomarkerEntry and BiomarkerDetail are modal presentations — they still need `useFocusEffect` for `setStatusBarStyle('dark')` since they present over the dark Dashboard tab.
- Settings and About are also modal presentations — same rule applies.

## React Native Implementation Notes

These rules apply to all modified files in this phase.

1. **StyleSheet only.** All styles in `const s = StyleSheet.create({...})` at bottom of file. No inline style objects except dynamic values (e.g. `{ color: someVar }`).

2. **Token imports.** Every file must import all tokens it uses: `import { Colors, Spacing, Radius, Typography, Elevation } from '../theme';`. Add `Elevation` to imports in files that currently spread shadow props manually.

3. **No hardcoded hex values.** Zero `'#...'` literals in any modified screen file. Use `Colors.Beige.*` or other named tokens only. This is a `tsc --noEmit` + grep-audited pass criterion.

4. **No hardcoded spacing numbers.** Zero literal margin/padding numbers. Use `Spacing.*` tokens only. Exception: `borderWidth: 0.5`, `borderWidth: 1`, height/width of decorative dots (8px dot = `width: 8, height: 8`) — these are fixed geometry values, not layout spacing.

5. **Elevation spread.** Replace all manually written `shadowColor/shadowOffset/shadowOpacity/shadowRadius/elevation` props with `...Elevation.sm` spread. Example:
   ```tsx
   card: {
     marginHorizontal: Spacing.base,
     backgroundColor: Colors.Beige.card,
     borderRadius: Radius.xl,
     ...Elevation.sm,
     overflow: 'hidden',
   }
   ```

6. **Status bar.** Use `setStatusBarStyle` from `expo-status-bar` (already a project dependency). Do NOT use the `<StatusBar>` JSX component — the `setStatusBarStyle` imperative API works correctly with React Navigation's focus lifecycle.

7. **useFocusEffect import.** Already imported in ExerciseScreen, ProfileScreen, ProtocolScreen, BiomarkerDetailScreen — reuse. Add to BiomarkerEntryScreen, SettingsScreen, AboutScreen (these currently lack it).

8. **No new packages.** This phase installs no new npm packages. All primitives are in the existing theme.

9. **TypeScript strict.** No `any`. All new variables must have inferred or explicit types. Empty state CTA callbacks must match existing navigation type `Nav`.

10. **Component size limit.** If any screen file exceeds 200 lines after changes, extract the empty state into a local component within the same file (no new file required — can be a local `function EmptyState() {...}` above the main export).
