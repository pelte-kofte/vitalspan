## Spacing Scale

All values from `Spacing.*` in `src/theme/index.ts`. Never hardcode margin/padding numbers.

| Token | Value | Usage |
|-------|-------|-------|
| `Spacing.xs` | 4 | Icon gaps, badge padding, inline micro-gaps |
| `Spacing.sm` | 8 | Between related elements (icon + label, checklist rows) |
| `Spacing.md` | 12 | Inner card padding (compact), between stacked labels |
| `Spacing.base` | 16 | Default horizontal screen padding, card inner padding |
| `Spacing.lg` | 20 | Between card sections, step indicator to card body |
| `Spacing.xl` | 24 | Between major card groups, section gaps |
| `Spacing.xxl` | 32 | Screen top padding, footer bottom padding |

Exceptions:
- Touch targets for primary CTA buttons: minimum 44pt height (iOS HIG) — use `paddingVertical: 16` inside a container with fixed `Spacing.base` horizontal padding to reach 48pt touch target height.
- Skip/defer link: minimum 44pt touch target via `paddingVertical: 12` on the touchable row.
- Step progress indicator top bar: `paddingTop: Spacing.xl` to clear the status bar on fullScreenModal.

## Typography

All values from `Typography.sizes.*` in `src/theme/index.ts`. Never hardcode font sizes.

| Role | Token | Size | Weight | Line Height |
|------|-------|------|--------|-------------|
| Body copy (card explanation) | `Typography.sizes.body` | 15px | 400 (regular) | 1.6 (24px) |
| Label / supporting text | `Typography.sizes.sm` | 12px | 400 (regular) | 1.4 |
| Card headline ("Why Glucose Matters") | `Typography.sizes.h3` | 18px | 600 (semibold) | 1.2 |
| Screen / section heading ("Step 1 of 3") | `Typography.sizes.lg` | 16px | 600 (semibold) | 1.2 |

Only two weights are used in this phase: **400 (regular)** for body copy and **600 (semibold)** for all headings, labels, and CTAs.

Letter spacing: use `Typography.letterSpacing.normal` (0) for body; `Typography.letterSpacing.wide` (0.5) for all-caps step progress labels.

Source: Established in existing screens (FutureSelf.tsx uses identical weight/size pattern).

## Color

All values from `Colors.*` in `src/theme/index.ts`. Never hardcode hex values in StyleSheet.create().

| Role | Token | Value | Usage |
|------|-------|-------|-------|
| Dominant (60%) | `Colors.bg` | `#EDE8DC` | Screen background, SafeAreaView fill |
| Secondary (30%) | `Colors.bgCard` | `#FFFFFF` | All card surfaces (explanation card, empty state card, FutureSelf wrapper) |
| Accent (10%) | `Colors.primary` | `#2D6A4F` | See reserved list below |
| Destructive | n/a | — | No destructive actions in this phase |

**Accent reserved for (explicit list):**
1. Primary CTA button background ("Log Glucose", "Log HbA1c", "Finish & See My Dashboard", "Log Your First Biomarkers", "Log Your First Result")
2. Step progress indicator active state (filled dot or active text)
3. Checkmark / completion indicator on FutureSelf locked checklist rows

Accent is NOT used for: card borders, body copy, icons, backgrounds, or secondary labels.

**Additional semantic color used:**
- Card border: `Colors.borderLight` (`#E2DED6`) — all card outlines in this phase
- Muted text: `Colors.textMuted` (`#8A8A82`) — skip link, helper labels, step counter "of 3"
- Body text: `Colors.textPrimary` (`#1A1A18`) — all readable copy
- Secondary body text: `Colors.textSecondary` (`#4A4A45`) — explanation card body copy (slightly softened)
- BreathingCard glow: `Colors.primaryDark` (`#1C3B2A`) — default `glowColor` prop (do not override)

Source: src/theme/index.ts, existing component patterns.

## Component Inventory

Components to create or modify in this phase:

### New: `GuidedFirstRunScreen` (`src/screens/GuidedFirstRunScreen.tsx`)

**Primary focal point:** the BreathingCard explanation card — the animated glow draws the eye first and establishes clinical context before the user interacts with the input field.

**Layout:**
- `SafeAreaView` with `backgroundColor: Colors.bg` — no NeuralGrid background (clean, focused)
- Top bar: `paddingTop: Spacing.xl`, centered step label + horizontal pill progress track
- Step progress: text label "Step 1 of 3" in `Typography.sizes.lg / weight 600 / Colors.textPrimary`, centered
- Below label: three dot indicators — active dot `Colors.primary` (10px diameter), inactive dots `Colors.border` (8px diameter), `gap: Spacing.sm`, centered row
- Main content: `ScrollView` with `paddingHorizontal: Spacing.base`
- Explanation card: `BreathingCard` wrapper (glowColor default `Colors.primaryDark`, period `Motion.breath`) containing a plain `View` with `backgroundColor: Colors.bgCard`, `borderRadius: Radius.lg`, `borderWidth: 1`, `borderColor: Colors.borderLight`, `padding: Spacing.base`
- Inside explanation card: clinical icon (see Copywriting Contract), headline (`Typography.sizes.h3 / weight 600 / Colors.textPrimary`), body copy (`Typography.sizes.body / weight 400 / Colors.textSecondary`, `lineHeight: 24`)
- Entry field: `TextInput` with `keyboardType: 'decimal-pad'`, height 52, `borderRadius: Radius.md`, `borderWidth: 1.5`, `borderColor: Colors.border` (focus state: `Colors.primary`), `fontSize: Typography.sizes.h3`, `fontWeight: '600'`, `color: Colors.textPrimary`, right-side unit label in `Colors.textMuted`
- RangeBar: render existing `RangeBar` component below the entry field — shows longevity target range visually before user types
- Footer: fixed-position footer with `paddingHorizontal: Spacing.base`, `paddingBottom: Spacing.xxl`, `paddingTop: Spacing.md`, `backgroundColor: Colors.bg`
  - Primary CTA button: `backgroundColor: Colors.primary`, `borderRadius: Radius.xl`, height 52, label in `Colors.bgCard / weight 600 / 16px`
  - Skip link: "I'll do this later" — centered, `Colors.textMuted / weight 400 / Typography.sizes.sm`, `paddingVertical: 12`

**State machine:** 3 steps via early-return pattern (`if (step === 0) return <StepZeroUI />`). Step 0 = Glucose, Step 1 = HbA1c, Step 2 = Cholesterol. Primary CTA label changes by step — see Copywriting Contract.

**Haptics:** `Haptics.notificationAsync(Haptics.NotificationFeedbackType.Success)` on final step completion. `Haptics.selectionAsync()` on step advance.

### Modified: `DashboardScreen` — Empty State Branch

**Condition:** `entries.length === 0`

**Empty state card layout:**
- Replace biomarker grid with single centered card
- Card: `backgroundColor: Colors.bgCard`, `borderRadius: Radius.lg`, `borderWidth: 1`, `borderColor: Colors.borderLight`, `marginHorizontal: Spacing.base`, `padding: Spacing.xl`
- Icon: centered, large (32px emoji or SVG)
- Heading: `Typography.sizes.h3 / weight 600 / Colors.textPrimary`, centered
- Body copy: `Typography.sizes.body / weight 400 / Colors.textSecondary`, centered, `lineHeight: 24`
- CTA button: same style as GuidedFirstRunScreen primary CTA — `backgroundColor: Colors.primary`, `borderRadius: Radius.xl`, height 48, full width within card padding
- `FutureSelf` component: rendered ABOVE the empty state card, in locked state (no props for biologicalAge/chronologicalAge — component handles locked rendering internally)

**Non-empty state:** no change to existing rendering.

### Modified: `BiomarkerDetailScreen` — Empty State

**Condition:** `entries.length === 0`

**Empty state layout:**
- Displayed where the biomarker list/grid normally appears
- Icon + heading + body + CTA button — same visual pattern as Dashboard empty state card
- Card container with same tokens: `Colors.bgCard`, `Radius.lg`, `Colors.borderLight`, `padding: Spacing.xl`
- No `FutureSelf` here (not relevant to Biomarkers tab context)

### Modified: `BiomarkerEntryScreen` — Explanation Card Insertion

**Condition:** `biomarkerId` is one of `['fastingglucose', 'hba1c', 'totalcholesterol']` AND content exists in `firstRunContent.ts`

**Insertion point:** Above the search/selector UI, when a biomarker with content is already selected
- Card visual: same BreathingCard + inner card pattern as GuidedFirstRunScreen explanation card
- Does NOT replace the existing UI — it prepends the explanation card to the existing flow

### New: `src/data/firstRunContent.ts`

Data structure (not UI, but consumed by UI components):

```typescript
interface FirstRunContent {
  biomarkerId: string;
  icon: string;           // single emoji, e.g. '🍬'
  headline: string;       // e.g. "Why Fasting Glucose Matters"
  body: string;           // 2-3 sentences, plain English, pharmacist-accurate
}
```

## Interaction Contracts

### GuidedFirstRunScreen

| Interaction | Behavior |
|-------------|---------|
| Tap "Log Glucose" / "Log HbA1c" | Validate input is non-empty and numeric. If invalid, show inline error. If valid: save entry to AsyncStorage `@vitalspan_biomarkers`, advance step counter, `Haptics.selectionAsync()` |
| Tap "Finish & See My Dashboard" | Save Cholesterol entry, write `@vitalspan_first_run_complete: true`, `Haptics.notificationAsync(Success)`, `nav.reset({ index: 0, routes: [{ name: 'Main' }] })` |
| Tap "I'll do this later" | Write `@vitalspan_first_run_complete: true`, `nav.reset({ index: 0, routes: [{ name: 'Main' }] })` — no confirmation dialog |
| TextInput focus | Border color transitions to `Colors.primary` — dynamic inline style |
| TextInput blur | Border color returns to `Colors.border` |
| Back gesture | Disabled — `presentation: 'fullScreenModal'` with `gestureEnabled: false` prevents accidental dismissal |

### Dashboard Empty State

| Interaction | Behavior |
|-------------|---------|
| Tap "Log Your First Biomarkers" | `nav.navigate('GuidedFirstRun')` |
| FutureSelf card in locked state | Tappable — shows the existing `showExplainer` modal explaining how projection unlocks |

### Biomarkers Tab Empty State

| Interaction | Behavior |
|-------------|---------|
| Tap "Log Your First Result" | `nav.navigate('GuidedFirstRun')` |
